# 好又来西餐厅微信点餐小程序 — 答辩文档

---

## 一、项目概述

本项目是一个基于微信小程序的餐厅点餐系统，包含用户端（微信小程序）和管理端（Vue 管理后台），后端采用 Node.js + Express 提供 RESTful API 服务。系统实现了菜单浏览、商品详情、购物车管理、订单支付流程、消息通知等核心功能，支持多人同 WiFi 环境下真机调试体验完整点餐流程。

```
│  │详情   │ │下单  │ │列表  │ │详情   │           │
│  └──────┘ └──────┘ └──────┘ └──────┘           │

┌─────────────────────────────────────────────────┐

│  ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│  │ 菜单接口  │ │ 购物车   │ │ 订单接口  │         │
│  │          │ │ 接口     │ │          │         │
│  └──────────┘ └──────────┘ └──────────┘         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│  │ 通知接口  │ │ 用户接口  │ │ 静态文件  │         │
│  │          │ │          │ │ (图片)   │         │
│  └──────────┘ └──────────┘ └──────────┘         │
└──────────────────┬──────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────┐
│               Vue 3 管理后台                       │
│           (Vite + Element Plus)                   │
└─────────────────────────────────────────────────┘
```

**数据流说明：**
1. 微信小程序通过 `wx.request` 向 Express 后端发送 HTTP 请求
2. GET 请求自动附加时间戳参数，防止浏览器缓存
3. 后端返回统一的 JSON 格式：`{ code: 0, data: {...} }` 或 `{ code: 1, message: '错误信息' }`
4. 前端根据 `code` 判断请求是否成功，成功则提取 `data` 字段使用
5. 图片资源通过 Express 静态文件服务 `/images` 提供

### 4.1 用户端（小程序）

| 模块 | 功能 | 对应页面 |
|---|---|---|
| 首页 | 品牌形象展示，进入店铺入口 | `pages/index/index` |

| 购物车 | 商品列表、数量增删、清空、合计计算、去结算 | `pages/cart/cart` |
| 结算下单 | 收货地址、商品清单、备注、订单提交 | `pages/checkout/checkout` |
| 订单列表 | 全部/待支付/已支付/已完成/已取消分类筛选 | `pages/order/order` |
| 订单详情 | 状态展示、收货信息、商品清单、金额明细、支付/取消 | `pages/order-detail/order-detail` |
| 消息通知 | 全部/优惠/订单/系统/公告分类，展开详情，已读/全部已读 | `pages/inform/inform` |
| 个人中心 | 用户信息、统计数据、功能入口（订单、优惠券、地址等） | `pages/mine/mine` |

#### 核心功能 — 菜单双栏联动

菜单页采用"左分类 + 右列表"的经典电商布局，核心逻辑：

- **点击联动：** 点击左侧分类项，右侧自动滚动到该分类对应区域（通过 `scroll-into-view` 实现）
- **滚动联动：** 右侧滚动时，通过监听 `bindscroll` 事件获取滚动位置，与预先计算的各分类标题距顶距离比对，自动高亮左侧对应分类
- **分类位置计算：** 在数据渲染完成后（`setData` 回调），使用 `wx.createSelectorQuery` 获取每个分类标题的 `boundingClientRect`，计算相对滚动视图顶部的偏移量

#### 核心功能 — 订单购买流程

```
菜单浏览 → 加入购物车 → 购物车管理 → 提交结算 → 创建订单 → 支付 → 订单完成
                                                                  ↓
                                                             取消订单
```

- 后端创建订单时自动计算满减优惠（满20减5，满30减10）
- 创建订单后自动清空购物车
- 支付和取消均为模拟操作，更新订单状态

## 五、后端 API 接口设计

```json
// 成功
```

## 七、前后端数据交互

### 7.1 交互方式

- 前端小程序通过 `wx.request` API 发起 HTTP 请求
- 后端 Express 服务器监听 `0.0.0.0:3000`，支持局域网访问
- 使用 CORS 中间件解决跨域问题
- 图片资源通过 `express.static` 静态文件中间件提供

### 7.2 前端请求封装（`utils/api.js`）

```javascript
function request(url, method = 'GET', data = {}) {
  // GET 请求加时间戳防缓存
  const ts = method === 'GET' ? '?_t=' + Date.now() : ''
  return new Promise((resolve, reject) => {
    wx.request({
      url: BASE_URL + url + ts,
      method, data,
      success: (res) => {
        if (res.data.code === 0) resolve(res.data.data)
        else reject(res.data)
      },
      fail: reject
    })
  })
}
```

### 7.3 数据交互示例（完整下单流程）

```
用户点击"提交订单"
    │
    ▼
checkout.js: onSubmit()
    │ POST /api/order/create { items, remark }
    ▼
server/index.cjs: 创建订单
    │ 1. 计算总价
    │ 2. 自动计算满减优惠
    │ 3. 生成订单对象
    │ 4. 清空购物车
    │ 5. 返回 { code: 0, data: { order } }
    ▼
checkout.js: 接收订单数据
    │ wx.redirectTo → /pages/order-detail/order-detail?id=xxx
    ▼
order-detail.js: onLoad
    │ GET /api/order/:id
    ▼
server: 返回订单详情
    │
    ▼
用户点击"立即支付"
    │ POST /api/order/:id/pay
    ▼
server: 更新订单状态为 paid
    │ 返回 { code: 0, message: '支付成功' }
    ▼
order-detail.js: 刷新订单状态
```

### 7.4 GET 请求缓存处理策略

为防止微信小程序对 GET 请求的缓存，每次请求自动附加 `_t=时间戳` 参数，确保每次请求都获取最新数据。

```javascript
const ts = method === 'GET' ? (url.includes('?') ? '&_t=' : '?_t=') + Date.now() : ''
```

## 八、项目运行说明

### 8.1 环境要求

- Node.js >= 22.18.0
- npm（Node.js 自带）
- 微信开发者工具（最新版本）

### 8.2 启动步骤

```bash
# 1. 进入后端目录
cd backend

# 2. 安装依赖
npm install

# 3. 启动 API 服务器（自动释放 3000 端口）
npm run api
# 输出：API 服务器已启动：http://localhost:3000
```

### 8.3 微信小程序

1. 打开微信开发者工具
2. 导入 `miniprogram/` 文件夹
3. 修改 `utils/api.js` 中的 `BASE_URL` 为本机局域网 IP（例如 `http://192.168.x.x:3000`）
4. 点击编译预览

### 8.4 真机调试

微信开发者工具 → 工具栏 → 真机调试 → 生成二维码 → 其他手机扫码（需在同一 WiFi 网络）

### 8.5 后端依赖说明（`package.json`）

```json
{
  "scripts": {
    "preapi": "node server/kill-port.cjs",   // 启动前释放 3000 端口
    "api": "node server/index.cjs"            // 启动 API 服务器
  }
}
```

端口释放脚本 `kill-port.cjs`：
- 通过 `netstat -ano | findstr :3000` 查找占用 3000 端口的进程 PID
- 通过 `taskkill /F /PID` 强制终止该进程
